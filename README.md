# wp-force-open-editor-metaboxes
Enable expanding and collapsing WP 7.0 metaboxes using mouse scrolling

# Wordpress Force Open Editor Metaboxes

WordPress 7.0 の記事編集画面で、カスタムフィールドなどのメタボックスが標準では隠れてしまう問題に対して、
WordPress 6.9.4 以前に近い操作感を再現するための対策プラグインです。

このプラグインを導入すると、Gutenberg 本文エディタとメタボックスの行き来を、
マウススクロール操作で行えるようになります。

> [!NOTE]
> スマートフォン版での動作は未確認です。

---

## 開発の背景

WordPress 6.9.4 までの環境では、記事編集画面で下にスクロールすると、
Gutenberg 本文エディタの下にカスタムフィールドなどのメタボックスが地続きに表示されていました。

<img width="942" height="183" alt="image" src="https://github.com/user-attachments/assets/76f20bea-80cc-4c6a-86df-d9c41a68db4b" />

しかし WordPress 7.0 では、メタボックス部分にハンドルのような UI が追加され、
ユーザーが明示的に展開しないとメタボックスが表示されない挙動になりました。

<img width="517" height="128" alt="image" src="https://github.com/user-attachments/assets/49403fdf-ef16-497d-b3b8-b053a3b50e1b" />

そのため、ACF などのカスタムフィールドを日常的に使っている環境では、
以下のような問題が起きる可能性があります。

* カスタムフィールドが消えたように見える
* 入力欄がどこにあるかわかりにくい
* WordPress に詳しくないユーザーから問い合わせが増える
* 従来の編集フローと大きく変わり、運用負荷が増える

この問題は WordPress 公式コミュニティでも話題になっています。

* https://wordpress.org/support/topic/ux-with-meta-boxes-hidden-by-default/
* https://wordpress.org/support/topic/disable-the-resize-handle-metabox/

そこで、この問題への暫定対応ができるプラグインを開発しました。

## できること

このプラグインをインストールすると、以下の操作が可能になります。

* 本文エディタを最下部までスクロールすると、メタボックスを自動展開する
* 本文エディタ最下部でさらに下方向にホイールすると、メタボックスを自動展開する
* メタボックス内の最上部で上方向にスクロールすると、メタボックスを自動格納する
* メタボックス格納後、本文エディタのスクロール位置を最下部へ戻す

これにより、WordPress 6.9.4 以前に近い感覚で、記事本文とカスタムフィールドを行き来できます。

> [!WARNING]
> 将来の WordPress アップデートにより、同様の挙動が本体側で改善された場合、このプラグインは不要になる可能性があります。

---

## インストール方法

このプラグインは通常のプラグインではなく、`mu-plugins` として設置する想定です。

### 1. `mu-plugins` ディレクトリを作成する

`/wp-content/` 内に `mu-plugins` ディレクトリが存在しない場合は作成してください。

```txt
wp-content/
    mu-plugins/
```

### 2. ファイルを配置する

以下のように配置してください。

#### 正しい配置

```txt
wp-content/
    mu-plugins/
        force-open-editor-metaboxes.php
        force-open-editor-metaboxes/
            script.js
```

#### 誤った配置

以下の配置では動作しません。

```txt
wp-content/
    mu-plugins/
        force-open-editor-metaboxes/
            index.php
            script.js
```

`mu-plugins` 配下のサブディレクトリ内に PHP ファイルを置いただけでは、WordPress が自動読み込みしないためです。

## アンインストール方法

`wp-content/mu-plugins/` 直下にある、
`force-open-editor-metaboxes/` ディレクトリと、
`force-open-editor-metaboxes.php` を削除してください。

`wp-content/mu-plugins/` 自体は、
本プラグイン以外に使用しているプラグインがなければ、削除しても構いません。

## 仕様

### 初期状態

記事編集画面を開いた直後は、必ずメタボックスを閉じた状態で開始します。
WordPress 7.0 の標準挙動では、
前回の開閉状態が引き継がれる場合がありますが、本プラグインでは引き継ぎません。

---

## メタボックスの展開

### トリガー

以下のいずれかの操作で、メタボックスを展開します。

* 記事本文を最下部までスクロールしたとき
* 記事本文の最下部で、さらに下方向にホイールしたとき

### 動作

メタボックス展開時は、以下の動作を行います。

* メタボックスを即時に展開する
* アニメーションは行わない
* 展開高さは編集エリアの約 45% に設定する
* 展開高さの上限は 50% とする
* ユーザーが手動でリサイズした高さは無視する
* 展開するたびに、目標高さへ正規化する
* 展開後、メタボックス内のスクロール位置を先頭にリセットする

### 制限

誤作動を防ぐため、以下の制限があります。

* ページ読み込み直後 1.2 秒間は自動展開しない
* メタボックス格納直後 0.45 秒間は再展開しない

これにより、初期化処理中の意図しない展開や、
閉じた直後にすぐ再展開される挙動を防ぎます。

---

## メタボックスの格納

### トリガー

以下のいずれかの操作で、メタボックスを格納します。

* メタボックス内で上方向にスクロールし、スクロール位置が最上部に到達したとき
* すでにメタボックス内の最上部にいる状態で、さらに上方向にホイールしたとき

### 動作

メタボックス格納時は、以下の動作を行います。

* メタボックスを即時に格納する
* アニメーションは行わない
* 格納後、本文エディタのスクロール位置を最下部に戻す

これにより、メタボックスを閉じた後も、本文エディタの続きから自然に作業できます。

### 例外

Advanced Custom Fields の `textarea` やリッチエディタ内でスクロールしている場合は、
誤ってメタボックスを格納しないように考慮しています。

---

## 想定用途

このプラグインは、以下のような環境を想定しています。

* WordPress 7.0 以降を使用している
* Gutenberg エディタを使用している
* ACF などのカスタムフィールドを記事編集画面で使用している
* WordPress 6.9.4 以前の編集画面に近い操作感を維持したい
* クライアントや運用担当者から「カスタムフィールドが見えなくなった」と問い合わせを受けている

---

## 注意事項

* WordPress 本体の管理画面 UI に依存しています
* WordPress の将来アップデートにより、動作しなくなる可能性があります
* スマートフォン版での動作は未確認です
* すべてのテーマ・プラグイン環境での動作を保証するものではありません
* ACF 以外のメタボックスについても対象になりますが、個別プラグイン固有の UI では意図しない挙動になる可能性があります
